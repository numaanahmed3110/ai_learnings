Tokens & the Context Window

This is the most practical LLM concept you'll learn all week. Tokens decide what you pay and what fits; the context window is the model's entire working memory. Miss this and your apps break in confusing ways.

Your win

You'll be able to estimate token counts and cost, explain what a context window is, and know why a chatbot "forgets" — three things you'll reason about constantly as an AI Engineer.

1A token is not a word

Last lesson we said an LLM predicts the next token. A token is a chunk of text — usually a piece of a word, not a whole word. Common words are often one token; longer or rarer words split into several; spaces and punctuation count too.1

Roughly, for English

1 token ≈ 4 characters ≈ ¾ of a word.
So 1,000 tokens ≈ 750 words, and a page of text is very roughly 500 tokens. "cat" is 1 token; "unbelievable" might be 3–4.1

You don't compute this by hand — but the rule of thumb lets you sanity-check sizes and bills in your head.

2Why tokens matter to you

  • Cost. Model APIs bill per token — both the tokens you send (input) and the tokens it generates (output), usually at different prices. A chatty prompt on a million requests is real money.2
  • Limits. Every model has a maximum number of tokens it can handle at once. Go over and you get an error or silently dropped text.
  • Speed. More tokens = more compute = slower and pricier. Trimming tokens is a core optimisation later (Phase 8).

3The context window = the model's RAM

The context window is the maximum number of tokens a model can consider at once — and it counts input + output together. Think of it as the model's working memory, or RAM: everything it can "see" for this one call has to fit inside it.3

CONTEXT WINDOW (e.g. 200,000 tokens)
system prompt + your instructions + retrieved docs + chat history + the model's reply — all must fit here

Modern windows are large (roughly 128K to 1M+ tokens depending on the model), but they are never infinite — and the whole window is re-read on every single call.

4The consequence that trips up beginners

The model has no memory of its own

An LLM does not remember your previous messages. A chatbot feels like it remembers only because the app re-sends the earlier conversation inside the context window every turn. Memory is engineered, not innate.3

Two big engineering ideas fall straight out of this:

  • A long document that doesn't fit must be split into chunks and only the relevant pieces fed in — the seed of RAG (retrieval-augmented generation), coming in Phase 7.
  • As a conversation grows, old turns must be summarised or dropped to stay within the window — the seed of agent memory.

5Do this now — 2 minutes, hands-on

See tokens with your own eyes

Open the OpenAI Tokenizer and paste a sentence. Watch it split into coloured tokens. Try unbelievable, a number like 1234567, an emoji, and a word in another language. Notice how token count ≠ word count. That felt sense is what you're after.1
